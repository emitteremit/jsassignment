Differences between VAR LET and CONST

1. Scope - *var*: Function-scoped. Declared variables are accessible within the function where they are defined. - *let & const*: Block-scoped. Only accessible within the {} block where they are defined.

2. Hoisting - *var*: Hoisted to the top of its scope and initialized with undefined. - *let & const*: Hoisted but not initialized (in temporal dead zone until declared).

3. Re-declaration - *var*: Can be re-declared within the same scope. - *let & const*: Cannot be re-declared in the same scope.

4. Reassignment - *var & let*: Can be reassigned. - *const*: Cannot be reassigned after initial assignment.

5. Global Object Property - *var* (in global scope): Becomes a property of the global object (window.varName). - *let & const*: Do not become global object properties.

6. Use in Loops - *var*: Does not create new scope per iteration. - *let*: Creates a new block scope for each iteration (e.g. in for loops).

7. Declaration Requirement - *var*: Can be declared without initialization. - *let*: Can be declared without initialization. - *const*: Must be initialized at declaration.

8. Mutability - All allow changing contents of objects/arrays (e.g. const obj = {}; obj.name = 'A' works). - *Only const* prohibits reassignment of the whole variable.

9. Practice

- *Avoid var* in modern JavaScript. - Use *let* for values that change. - Use *const* by default unless you need reassignment.

10. Temporal Dead Zone (TDZ) - *var*: No TDZ, you get undefined if accessed before declaration. - *let & const*: TDZ exists — accessing before declaration throws ReferenceError.
